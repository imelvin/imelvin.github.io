本文基于JDK 1.7的源码。

接口

首先我们看线程池实现的接口。

Executor接口

该接口的方法是线程池常用的一个方法。一般来说，线程池需要将任务封装成Runnable对象来执行。该接口将任务与任务的执行策略去耦合，任务由Runnable实现，而任务的执行策略由Executor的实现类负责。

如果直接使用线程，就需要自己处理任务的调度，执行等等。为了重用这些策略，我们可以将这些策略封装成Executor对象。

ExecutorService接口

该接口是Executor接口的扩展，ThreadPoolExecutor类最终实现的就是这个接口。

该接口提供了关闭线程池的方法，并提供了返回Future对象的任务提交方法，可以用来追踪异步任务的执行情况。

接口方法的作用如下：

  • shutdown 平缓关闭线程池，停止接受新任务，但是会将之前提交的任务执行完
  • shutdownNow 尝试立即关闭线程池，并将未执行的任务返回
  • isShutdown 线程池是否已经关闭
  • isTerminated 线程池关闭后是否所有任务都执行完成
  • awaitTermination 阻塞等待所有任务执行完成
  • submit 几个submit方法都是提交任务并返回一个Future对象用来获取该任务的执行结果和管理任务执行状态
  • invokeAll 批量提交任务，并批量返回Future对象
  • invokeAny 批量提交任务，返回一个执行完成的结果

AbstractExecutorService实现

该类是一个抽象类，提供了一些ExecutorService的默认实现。

newTaskFor方法

首先AbstractExecutorService实现了两个newTaskFor方法，这两个方法把Runnable和Callable包装为FutureTask，这是一个Future对象，可以获取执行结果和管理执行状态。

submit方法

submit方法首先检查参数，然后通过newTaskFor方法将提交的任务包装为RunnableFuture对象，RunnableFuture既是一个Future对象又是一个Runnable对象，然后将该对象通过execute提交到线程池执行，并返回RunnableFuture对象给调用者。

AbstractExecutorService类中并没有实现execute方法。

invokeAny方法

两个invokeAny都是调用的doInvokeAny方法。

doInvokeAny方法还是首先检查参数，然后将当前对象包装为一个ExecutorCompletionService对象。然后先提交一个任务开始执行，再开始循环，如果某个任务执行成功，会返回结果，如果所有任务都执行异常，那么会抛出最后一次捕获的异常。方法返回之前，会取消所有正在执行的任务。

invokeAll方法

invokeAll方法比较简单，就是提交所有任务，然后等待所有任务执行完成，最终返回Future对象的集合。最终，如果有任何未被捕获的异常，在抛出之前会关闭所有已提交的任务。

ThreadPoolExecutor实现

control state

首先ThreadPoolExecutor有一个控制字段ctl和ctl相关的工具方法。

ctl是一个AtomicInteger，AtomicInteger是一个基于CAS实现的原子整数类。
ctl这个整数里面前3位是runState，表示线程池的运行状态，后29位是workerCount，表示工作线程的数量。

构造方法

构造方法负责设置一些参数。

这些参数作用如下：

  • corePoolSize 线程池保持的线程数量，除非allowCoreThreadTimeOut设置为真，否则这些线程将一直存活，哪怕没有任务
  • maximumPoolSize 线程池最大允许的线程数量
  • keepAliveTime 当线程数量大于corePoolSize时，多余的线程等待任务的时间，当core pool以外的线程等待任务超过这个时间就会终结
  • unit keepAliveTime的时间单位
  • workQueue 保存execute方法提交任务的队列
  • threadFactory 线程工厂类，executor创建线程使用
  • handler 阻塞处理器，处理由于工作队列满造成的执行阻塞

execute方法

execute方法先取ctl字段，判断工作线程数量小于corePoolSize时，就添加一个工作线程，并以刚提交的任务为第一个任务，然后获取最新的ctl字段。如果线程池仍在运行就讲command添加到workQueue，此时再检查一下ctl，如果线程池已经停止运行就将command从workQueue中移除并拒绝这次提交，当线程池还在运行但是工作线程都结束了就添加新的工作线程。当command添加到工作队列失败而添加工作线程也失败，则拒绝这次提交。

从代码分析

  1. 提交任务时会先创建线程来执行任务
  2. 当线程数量达到corePoolSize时，将任务放入workQueue中
  3. 当workQueue满后再重新创建线程
  4. 当线程数量达到maximumPoolSize时，再提交任务会被拒绝

addWorker方法

该方法是ThreadPoolExecutor类的核心方法之一，用途是添加工作线程。

首先是利用CAS操作非阻塞地更新workerCount，假如更新过程中线程池关闭了或者线程数量超过限制则返回添加失败。

第二步是创建Worker对象，也就是工作线程。工作线程保存在一个HashSet中

由于HashSet并非线程安全，而且workers的可能在多线程中更新，所以使用了ReentrantLock来保证线程安全

创建好Worker对象以后，获取锁，然后将Worker的线程放入workers集合，并最终启动该线程。

最终会检查线程workerStarted字段，如果该字段不为true，说明线程池正在关闭，那么就会调用addWorkerFailed方法回滚。

addWorkerFailed方法会清除新添加的Worker对象，再将workerCount数量减少，最后尝试终止线程池。

Worker类

Worker代表一个工作者，每个工作者代表一个工作线程

Worker类使用AbstractQueuedSynchronizer实现成了一个简单的互斥锁,用以在执行task时加锁，以免在任务执行过程中被中断。

runWorker方法

该方法是Worker的执行循环，不断通过getTask从workQueue获取任务执行。

getTask方法

该方法负责从workQueue中取task以供Worker执行。如果返回null，则调用getTask的Worker线程会结束。当allowCoreThreadTimeOut为false，而且Worker是core Worker时会调用take方法阻塞等待直到队列返回task；其他情况都是调用poll方法等待keepAliveTime的时间，然后返回null结束该线程。

shutdown方法

该方法将平缓关闭线程池。

调用该方法会立即返回，线程池状态变为SHUTDOWN，将不再接受新任务，而且会关掉空闲的工作线程，但是之前提交的任务会一直执行完成。

shutdownNow方法

该方法将立即关闭线程池。该方法尝试立即关闭所有工作线程，并将尚未执行的任务返回。但是正在执行的任务还是会继续执行完。

总结

线程池实际上是一个生产者-消费者模型。每个调用者都是一个生产者，而Worker线程就是消费者，workQueue就是共享的工作队列。每个要使用线程池的调用者将任务放入workQueue，而Worker线程不断从workQueue取任务执行。
